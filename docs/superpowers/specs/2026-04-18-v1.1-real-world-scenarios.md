# v1.1 Real-World Scenarios — 160 Institutional-Grade Improvements

**Date:** 2026-04-18
**Author:** 13-persona audit team, each wearing a senior FX-trader hat
**Purpose:** Expand v1.1 from "fix gaps in v1.0.3" to "software a 50-year institution
would ship and that makes profitable traders choose it over Edgewonk / TradeZella /
TraderVue." Every scenario below is a *real situation* a working trader faces. Not
a feature list — a scenario list. Each one answers: *"When does the trader open
the app and what do they actually need to do?"*

> **User's seed scenario (#1):** "A connected account doesn't just see trades — it
> also sees the income and outgoings to stay balanced and reconciled with MT5
> locally." That single scenario is the spine. Every scenario below is the kind of
> thing v1.1 should absorb so the app is **1000× v1.0.3 without crashing the
> update**.

---

## How to read this

- **Categories** group scenarios by capability area, but a scenario can satisfy
  multiple personas.
- **Priority labels** (P0 = v1.1 must-have, P1 = v1.1 should-have, P2 = v1.2,
  P3 = v2.0) map directly into the Option C 28-day plan. Final scoping happens
  in the plan doc; this doc captures the *universe* to choose from.
- **Effort:** S = 1 day, M = 2-4 days, L = 1-2 weeks, XL = 2+ weeks.

## 1. Account as full ledger (not just a trade list) — **P0**

*The seed scenario. Foundation for everything else.*

1. **S01 — Capture deposits + withdrawals from MT5 bridge.** New `balance_operations`
   table. EA extended to emit `BALANCE`/`CREDIT`/`CHARGE`/`CORRECTION`/`BONUS` deal
   events. — P0, L
2. **S02 — Balance reconciliation check on every sync.** App computes
   `starting + Σ(deposits) − Σ(withdrawals) + Σ(trade_net_pnl) + Σ(swaps) +
   Σ(commissions) + Σ(adjustments)` and compares to broker-reported balance. Red
   banner if drift > 0.01. — P0, M
3. **S03 — Missing-trade detector.** If balance drift exists and no balance op
   explains it, flag "broker balance changed by $X on YYYY-MM-DD — no matching
   event." — P0, S
4. **S04 — Deposit/withdrawal log UI.** Account page shows full cash-flow ledger
   separate from trades. — P0, M
5. **S05 — True equity curve (Modified Dietz).** Dashboard equity line excludes
   deposits/withdrawals so curve reflects *skill*, not capital additions. — P0, M
6. **S06 — Money-weighted return (IRR).** Annualized return accounting for
   mid-period cash flows. — P1, M
7. **S07 — Time-weighted return (GIPS-compliant).** Sub-period chaining so
   deposits don't inflate measured performance. — P1, M
8. **S08 — Peak balance tracking.** Max equity ever seen per account — for
   drawdown-from-peak prop firm rules. — P0, S
9. **S09 — Deposit history chart overlay on equity curve.** Vertical lines at
   each deposit with tooltip showing amount. — P1, S
10. **S10 — Broker fee breakdown widget.** Dashboard card: swaps paid,
    commissions paid, adjustments received, net fee drag in bps/year. — P1, M

## 2. Prop firm compliance — **P0**

*FTMO, MFF, Topstep, The5ers, E8, FundedNext, MyForexFunds, etc. Each has its own
daily loss / max loss / min days / consistency / news trading rules. Getting any
of these wrong kills the account.*

11. **S11 — Prop firm preset library.** Dropdown: FTMO Challenge $100k, FTMO Swing,
    MFF Rapid, Topstep TCP, E8, FundedNext Stellar, etc. Each preset populates
    daily loss %, max DD %, profit target %, min days, news rules, consistency
    rules. — P0, M
12. **S12 — Daily loss circuit breaker.** When realized + floating P&L hits
    daily loss %, app shows fullscreen red modal: "DAILY LOSS BREACH — STOP
    TRADING NOW." — P0, S
13. **S13 — Pre-trade rule check.** Before accepting a manual trade entry,
    simulate: "this trade at max SL would put daily loss at X% — breach=Y/N." — P0, M
14. **S14 — News-trading-forbidden window.** Most prop firms ban trading ±2-5
    minutes around red-impact news. App must block entry (or warn) inside the
    window based on connected calendar. — P0, M
15. **S15 — Consistency rule tracker.** "No single day > 30% of total profit"
    (FTMO) etc. — widget shows which days are concentration risks. — P1, M
16. **S16 — Minimum trading days tracker.** "Must trade N days before passing" —
    countdown + tick-off per day. — P0, S
17. **S17 — Weekend/holiday close-all enforcement.** Prop rules that require
    flat by Friday close — reminder 30 minutes before session close. — P1, S
18. **S18 — Stop-loss-required check.** Many prop firms reject trades without a
    stop. Flag at entry. — P0, S
19. **S19 — Maximum lot / leverage enforcement.** Firm-specific caps. — P0, S
20. **S20 — Evaluation → Funded phase migration.** When user passes, app
    imports new funded account as a linked continuation. — P1, M
21. **S21 — Scaling plan tracker.** Funded accounts often scale at profit
    milestones — widget shows next scale-up threshold. — P2, M
22. **S22 — Payout tracker.** Log each withdrawal from funded account as a
    "payout" with screenshot for audit. — P1, S

## 3. Risk & position sizing — **P0/P1**

*Every trader's first mistake is position-sizing error. Second is revenge sizing.

27. **S27 — Pyramid calculator.** "Add to winner at +1R with 0.5× size" —

28. **S28 — Break-even trigger reminder.** Post-entry, if price hits +1R and SL
    hasn't moved to BE, nudge user. — P1, M

30. **S30 — Kelly criterion per setup.** Given win rate + avg R, optimal bet

31. **S31 — Fractional Kelly (25% Kelly) as default sizing mode.** — P2, S

34. **S34 — Margin usage widget.** Free margin %, used margin %, margin call

## Cross-cutting observations

- **20+ scenarios depend on the balance_operations table (S01-S10, S25, S91-S96,
  S97, S102).** This is the single biggest architectural dependency in v1.1.
  Ship it week 1.
- **15+ scenarios depend on a real setup library (S68-S76, S60, S63).** Ship
  it week 2 so analytics can index against it in weeks 3-4.
- **18+ scenarios depend on live bridge enhancements (S01, S14, S35, S36-S38,
  S45, S48, S49, S81).** EA changes required — revise `LedgerBridge.mq5` once
  and ride the wave.
- **12+ scenarios depend on the news/event calendar (S47-S55, S14, S50, S51).**
  Auto-sync calendar early so downstream features have data.
- **Security P0s (S107-S110, S114) are independent.** Can ship in parallel.
